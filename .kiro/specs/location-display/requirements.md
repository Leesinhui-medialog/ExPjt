# 요구사항 문서

## 소개

사용자의 현재 위치 정보를 브라우저 Geolocation API를 통해 조회하고, 지도와 함께 화면에 표시하는 기능이다.
로그인한 사용자가 좌측 메뉴에서 "위치 정보" 메뉴를 선택하면, 브라우저에서 위치 권한을 요청하고 현재 위도·경도를 기반으로 지도 및 주소 정보를 표시한다.
또한 Kakao 로컬 API로 수집하여 JSON 파일로 저장한 전국 골프장 위치 데이터를 백엔드 API에서 읽어 지도에 마커로 표시하고, 지도 좌측의 검색 패널을 통해 골프장을 이름이나 지역으로 검색할 수 있다.

## 용어 정의

- **위치_정보_페이지**: 사용자의 현재 위치를 지도와 텍스트로 표시하는 React 페이지 컴포넌트
- **Geolocation_API**: 브라우저에서 제공하는 `navigator.geolocation` API로, 사용자의 현재 위도·경도를 반환한다
- **위치_정보_API**: 백엔드에서 위도·경도를 받아 주소 정보로 변환(역지오코딩)하여 반환하는 REST API
- **좌측_메뉴**: 애플리케이션의 공통 네비게이션 메뉴 컴포넌트 (LeftMenu)
- **위치_정보**: 위도(latitude), 경도(longitude), 주소(address) 등을 포함하는 데이터 객체
- **골프장_API**: 백엔드에서 정적 JSON 파일을 읽어 전국 골프장 목록을 조회·검색하여 반환하는 REST API
- **골프장_데이터**: 골프장 이름(name), 위도(latitude), 경도(longitude), 주소(address), 지역(region), 유형(type: 회원제/퍼블릭/회원제+퍼블릭) 등을 포함하는 데이터 객체
- **골프장_JSON_파일**: Kakao 로컬 API로 수집한 전국 골프장 데이터를 저장한 정적 JSON 파일 (`golf-courses.json`)
- **골프장_수집_스크립트**: Kakao 로컬 API를 호출하여 전국 골프장 데이터를 수집하고 JSON 파일로 저장하는 Node.js 스크립트
- **범례**: 검색_패널 내에 표시되는 마커 색상별 골프장 유형 안내 영역
- **검색_패널**: 지도 위에 레이어(오버레이) 형태로 좌측에 떠 있는 반투명 패널로, 골프장을 이름 또는 지역으로 검색할 수 있는 UI 영역
- **골프장_마커**: 지도 위에 골프장 위치를 표시하는 마커 아이콘으로, 골프장 유형에 따라 색상이 구분된다 (회원제: 파란색, 퍼블릭: 초록색, 회원제+퍼블릭: 오렌지색)
- **네이버_길찾기**: 네이버 지도의 길찾기 서비스로, URL 파라미터를 통해 출발지와 도착지를 지정하여 경로를 안내하는 웹 페이지
- **길찾기_버튼**: 골프장_마커 팝업 내에 표시되는 버튼으로, 클릭 시 네이버_길찾기 페이지를 새 탭으로 연다

## 요구사항

### 요구사항 1: 위치 정보 메뉴 추가

**사용자 스토리:** 사용자로서, 좌측 메뉴에서 위치 정보 페이지로 이동할 수 있기를 원한다. 그래야 쉽게 위치 정보 기능에 접근할 수 있다.

#### 인수 조건

1. THE 좌측_메뉴 SHALL "위치관리" 메뉴 그룹 아래에 "위치 정보" 하위 메뉴 항목을 표시한다
2. WHEN 사용자가 "위치 정보" 메뉴를 클릭하면, THE 좌측_메뉴 SHALL `/location/display` 경로로 페이지를 이동시킨다
3. WHILE 사용자가 `/location/display` 경로에 있는 동안, THE 좌측_메뉴 SHALL 해당 메뉴 항목을 활성 상태로 표시한다

### 요구사항 2: 브라우저 위치 권한 요청 및 좌표 조회

**사용자 스토리:** 사용자로서, 위치 정보 페이지에 진입하면 자동으로 현재 위치를 조회할 수 있기를 원한다. 그래야 별도 조작 없이 위치를 확인할 수 있다.

#### 인수 조건

1. WHEN 위치_정보_페이지가 로드되면, THE 위치_정보_페이지 SHALL Geolocation_API를 통해 브라우저에 위치 권한을 요청한다
2. WHEN Geolocation_API가 위도와 경도를 반환하면, THE 위치_정보_페이지 SHALL 위도와 경도 값을 화면에 텍스트로 표시한다
3. WHILE Geolocation_API 응답을 대기하는 동안, THE 위치_정보_페이지 SHALL "위치 정보를 조회 중입니다" 로딩 메시지를 표시한다
4. IF 사용자가 위치 권한을 거부하면, THEN THE 위치_정보_페이지 SHALL "위치 권한이 거부되었습니다. 브라우저 설정에서 위치 권한을 허용해 주세요." 안내 메시지를 표시한다
5. IF 브라우저가 Geolocation_API를 지원하지 않으면, THEN THE 위치_정보_페이지 SHALL "이 브라우저는 위치 정보를 지원하지 않습니다." 안내 메시지를 표시한다

### 요구사항 3: 역지오코딩을 통한 주소 표시

**사용자 스토리:** 사용자로서, 위도·경도뿐 아니라 사람이 읽을 수 있는 주소 형태로 현재 위치를 확인하고 싶다. 그래야 위치를 직관적으로 이해할 수 있다.

#### 인수 조건

1. WHEN 위도와 경도가 조회되면, THE 위치_정보_페이지 SHALL 위치_정보_API에 위도와 경도를 전달하여 주소 정보를 요청한다
2. WHEN 위치_정보_API가 주소를 반환하면, THE 위치_정보_페이지 SHALL 주소를 화면에 텍스트로 표시한다
3. THE 위치_정보_API SHALL `/api/location/reverse-geocode` 경로로 GET 요청을 수신하고, 위도(lat)와 경도(lng) 쿼리 파라미터를 받아 주소 문자열을 JSON 형태로 반환한다
4. IF 위치_정보_API가 주소 변환에 실패하면, THEN THE 위치_정보_페이지 SHALL "주소 정보를 가져올 수 없습니다." 안내 메시지를 표시한다

### 요구사항 4: 지도에 현재 위치 표시

**사용자 스토리:** 사용자로서, 현재 위치를 지도 위에 마커로 확인하고 싶다. 그래야 시각적으로 위치를 파악할 수 있다.

#### 인수 조건

1. WHEN 위도와 경도가 조회되면, THE 위치_정보_페이지 SHALL 지도 영역에 해당 좌표를 중심으로 지도를 표시한다
2. THE 위치_정보_페이지 SHALL 현재 위치에 마커를 표시한다
3. THE 위치_정보_페이지 SHALL 지도 영역의 너비를 부모 컨테이너의 100%, 높이를 400px로 설정한다
4. WHEN 사용자가 마커를 클릭하면, THE 위치_정보_페이지 SHALL 마커 위에 주소 정보를 팝업으로 표시한다

### 요구사항 5: 위치 새로고침

**사용자 스토리:** 사용자로서, 위치가 변경되었을 때 수동으로 위치를 다시 조회할 수 있기를 원한다. 그래야 이동 후에도 최신 위치를 확인할 수 있다.

#### 인수 조건

1. THE 위치_정보_페이지 SHALL "위치 새로고침" 버튼을 표시한다
2. WHEN 사용자가 "위치 새로고침" 버튼을 클릭하면, THE 위치_정보_페이지 SHALL Geolocation_API를 다시 호출하여 현재 위치를 재조회한다
3. WHEN 위치가 재조회되면, THE 위치_정보_페이지 SHALL 지도 중심 좌표, 마커 위치, 위도·경도 텍스트, 주소 정보를 갱신한다

### 요구사항 6: 인증된 사용자만 접근 허용

**사용자 스토리:** 관리자로서, 로그인한 사용자만 위치 정보 페이지에 접근할 수 있기를 원한다. 그래야 인증되지 않은 접근을 방지할 수 있다.

#### 인수 조건

1. WHEN 로그인하지 않은 사용자가 `/location/display` 경로에 접근하면, THE 위치_정보_페이지 SHALL 사용자를 `/login` 경로로 리다이렉트한다
2. THE 위치_정보_API SHALL 세션 기반 인증을 확인하고, 인증되지 않은 요청에 대해 HTTP 401 상태 코드를 반환한다


### 요구사항 7: 전국 골프장 데이터 조회 및 지도 마커 표시

**사용자 스토리:** 사용자로서, 전국 골프장의 위치를 지도에서 한눈에 확인하고 싶다. 그래야 원하는 골프장의 위치를 시각적으로 파악할 수 있다.

#### 인수 조건

1. WHEN 위치_정보_페이지가 로드되면, THE 위치_정보_페이지 SHALL 골프장_API를 호출하여 전국 골프장 목록을 조회한다
2. WHEN 골프장_API가 골프장 목록을 반환하면, THE 위치_정보_페이지 SHALL 각 골프장의 위도·경도를 기반으로 지도에 골프장_마커를 표시한다
3. THE 위치_정보_페이지 SHALL 회원제 골프장의 골프장_마커를 파란색으로 표시한다
4. THE 위치_정보_페이지 SHALL 퍼블릭 골프장의 골프장_마커를 초록색으로 표시한다
5. THE 위치_정보_페이지 SHALL 회원제와 퍼블릭 모두 해당하는 골프장의 골프장_마커를 오렌지색으로 표시한다
6. THE 골프장_API SHALL `/api/golf-courses` 경로로 GET 요청을 수신하고, 골프장_JSON_파일을 읽어 전국 골프장 목록을 JSON 배열 형태로 반환한다
7. THE 골프장_API SHALL 각 골프장 데이터에 골프장 이름, 위도, 경도, 주소, 지역, 유형(회원제/퍼블릭/회원제+퍼블릭) 정보를 포함하여 반환한다
8. THE 골프장_수집_스크립트 SHALL Kakao 로컬 API의 키워드 검색을 사용하여 "골프장" 키워드로 전국 골프장 데이터를 수집한다
9. THE 골프장_수집_스크립트 SHALL 수집한 데이터를 골프장_JSON_파일(`src/main/resources/data/golf-courses.json`)로 저장한다
8. WHEN 사용자가 골프장_마커를 클릭하면, THE 위치_정보_페이지 SHALL 마커 위에 골프장 이름, 주소, "길찾기" 버튼을 포함한 팝업을 표시한다
9. IF 골프장_API 호출이 실패하면, THEN THE 위치_정보_페이지 SHALL "골프장 정보를 불러올 수 없습니다." 안내 메시지를 표시한다

### 요구사항 8: 골프장 검색 패널

**사용자 스토리:** 사용자로서, 지도 좌측의 검색 패널에서 골프장을 이름이나 지역으로 검색하고 싶다. 그래야 원하는 골프장을 빠르게 찾을 수 있다.

#### 인수 조건

1. THE 위치_정보_페이지 SHALL 지도 영역 위에 레이어(오버레이) 형태로 검색_패널을 좌측에 띄워서 배치한다
2. THE 검색_패널 SHALL position absolute 또는 fixed 방식으로 지도 위에 겹쳐 표시되며, 반투명 배경을 적용하여 지도가 부분적으로 보이도록 한다
2. THE 검색_패널 SHALL 골프장 이름 또는 지역을 입력할 수 있는 검색 입력 필드를 표시한다
3. WHEN 사용자가 검색어를 입력하면, THE 검색_패널 SHALL 골프장_API에 검색어를 전달하여 일치하는 골프장 목록을 조회한다
4. THE 골프장_API SHALL `/api/golf-courses/search` 경로로 GET 요청을 수신하고, keyword 쿼리 파라미터를 받아 골프장 이름 또는 지역에 해당 키워드가 포함된 골프장 목록을 JSON 배열로 반환한다
5. WHEN 골프장_API가 검색 결과를 반환하면, THE 검색_패널 SHALL 검색 결과 목록을 골프장 이름과 주소 형태로 표시한다
6. IF 검색 결과가 없으면, THEN THE 검색_패널 SHALL "검색 결과가 없습니다." 안내 메시지를 표시한다
7. WHILE 검색_패널의 검색어가 비어 있는 동안, THE 검색_패널 SHALL 전체 골프장 목록을 표시한다
8. THE 검색_패널 SHALL 하단 또는 상단에 범례를 표시하여, 파란색 마커는 "회원제", 초록색 마커는 "퍼블릭", 오렌지색 마커는 "회원제+퍼블릭"임을 안내한다

### 요구사항 9: 검색 결과 선택 시 지도 이동 및 마커 강조

**사용자 스토리:** 사용자로서, 검색 결과에서 골프장을 선택하면 해당 위치로 지도가 이동하고 마커가 강조되기를 원한다. 그래야 선택한 골프장의 위치를 즉시 확인할 수 있다.

#### 인수 조건

1. WHEN 사용자가 검색_패널의 검색 결과 목록에서 골프장 항목을 클릭하면, THE 위치_정보_페이지 SHALL 해당 골프장의 위도·경도를 중심으로 지도를 이동한다
2. WHEN 지도가 선택된 골프장 위치로 이동하면, THE 위치_정보_페이지 SHALL 해당 골프장의 골프장_마커를 강조 표시한다
3. WHEN 골프장_마커가 강조되면, THE 위치_정보_페이지 SHALL 해당 마커의 팝업을 자동으로 열어 골프장 이름, 주소, "길찾기" 버튼을 표시한다
4. THE 위치_정보_페이지 SHALL 지도 이동 시 적절한 줌 레벨(15 이상)로 확대하여 선택된 골프장 위치를 명확히 표시한다

### 요구사항 10: 네이버 길찾기 연동

**사용자 스토리:** 사용자로서, 골프장 마커 팝업에서 "길찾기" 버튼을 클릭하면 네이버 지도 길찾기로 이동하고 싶다. 그래야 현재 위치에서 골프장까지의 경로를 바로 확인할 수 있다.

#### 인수 조건

1. WHEN 사용자가 골프장_마커 팝업의 길찾기_버튼을 클릭하면, THE 위치_정보_페이지 SHALL 네이버_길찾기 페이지를 새 브라우저 탭으로 연다
2. THE 위치_정보_페이지 SHALL 네이버_길찾기 URL의 출발지를 Geolocation_API로 조회한 사용자의 현재 위도·경도로 설정한다
3. THE 위치_정보_페이지 SHALL 네이버_길찾기 URL의 도착지를 해당 골프장의 위도·경도로 설정한다
4. THE 위치_정보_페이지 SHALL 네이버_길찾기 URL의 경로 유형을 자가용(car)으로 기본 설정하여, `https://map.naver.com/p/directions/{출발위도},{출발경도},{도착위도},{도착경도}/-/car` 형식으로 구성한다
5. IF 사용자의 현재 위치가 조회되지 않은 상태에서 길찾기_버튼을 클릭하면, THEN THE 위치_정보_페이지 SHALL "현재 위치를 먼저 조회해 주세요." 안내 메시지를 표시한다
